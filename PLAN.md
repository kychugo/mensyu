## 一、現有版本分析 / Existing Versions Analysis

| 版本 | 主要特色 |
|------|----------|
| `v1.HTML` | 基礎翻譯工具，點擊展開解釋 |

| `v3.HTML` | 全平台：翻譯、問答、3D磚塊遊戲、社交、古人角色系統 |

## 二、統一版本目標 / Unified Version Goals

將所有版本的最佳功能整合為一個單一 HTML 檔案，圍繞以下主線：

```
挑戰問答 → 翻譯文言文 → 與古人聊天 → 小遊戲獎勵
Challenge Q&A → Translate Classical Chinese → Chat with Ancients → Mini-Game Rewards
```

### 3.2 頁面結構 / Page Structure

統一版本採用**單一 HTML 多頁面架構**，共 5 個主要頁面（section）：

#### 頁面 1：🏠 首頁 (Home)
- **角色卡片牆**：顯示所有可解鎖的古代文豪
  - 🔒 鎖定狀態：顯示名稱、朝代、代表作
  - ✅ 解鎖狀態：顯示頭像、解鎖日期、對話紀錄數
- **學習進度總覽**：已解鎖角色數 / 總角色數，總積分
- **快速入口**：「繼續學習」按鈕，直接進入上次的進度

#### 頁面 2：📚 文章閱讀 (Read & Translate)
- **文章選擇器**：根據已解鎖角色顯示對應文章（難度分級 ⭐ ~ ⭐⭐⭐⭐⭐）
- **翻譯工作區**（來自 `v2.html` + `mensyu-tran.html`）：
  - 左欄：原文輸入 / 顯示
  - 右欄：AI 翻譯輸出
  - 逐字解釋（點擊字符展開）
  - 浮動原文視窗（可拖曳）
  - 「全部展開」按鈕
- **閱讀完成按鈕**：觸發進入問答關卡

#### 頁面 3：❓ 問答挑戰 (Quiz)
- **問題顯示區**（來自 `mensyu2.html`）：
  - 當前問題 / 總問題（進度條動畫）
  - 問題文字
  - 4 個選項按鈕（A/B/C/D）
  - 即時反饋動畫（綠色✅ / 紅色❌）
- **結果頁面**：
  - 成功（≥ 4/5）：顯示分數 + 解鎖動畫 + 進入聊天按鈕
  - 失敗（< 4/5）：顯示錯誤分析 + 重試按鈕 + 返回閱讀按鈕
- **額外挑戰**（進階功能）：
  - 填空題模式：在文章中找出關鍵字
  - 排序題：將打散的句子重新排列

#### 頁面 4：💬 與古人聊天 (Chat with Ancients)
- **古人對話界面**（全新設計，整合 `v3.HTML` 社交功能）：
  - 頂部顯示古人頭像、姓名、朝代
  - 聊天氣泡界面（古人訊息 ← 左側，用戶訊息 → 右側）
  - 輸入框 + 發送按鈕
  - **話題建議卡片**：推薦問題（如「請問您為何寫〈赤壁賦〉？」）
  - **古人主動發言**：古人會定時分享詩句或見解
- **文言文模式**（進階）：
  - 切換按鈕：白話文 ⇌ 文言文模式
  - 文言文模式：古人以文言文回覆，並附白話翻譯
- **聊天紀錄**：儲存至 localStorage，可隨時回顧

#### 頁面 5：🎮 小遊戲大廳 (Game Hall)
- 顯示所有可用小遊戲（見 3.3）
- 每個遊戲顯示：最高分數、解鎖要求、難度

### 3.3 小遊戲設計 / Mini-Games Design

#### 遊戲一：🧱 詩詞磚塊 (Poetry Brick Breaker)
> **來源**：整合自 `v3.HTML` + `mensyu2.html` 的 Three.js 3D 磚塊遊戲

- **玩法**：3D 磚塊打磚塊，每個磚塊上印有文言文字符
- **學習元素**：打碎磚塊時顯示該字的解釋
- **積分系統**：分數越高，獲得越多學習積分
- **控制方式**：電腦（← → 鍵盤），手機（左右觸控板）

#### 遊戲二：✍️ 默寫挑戰 (Dictation Challenge)
> **全新設計**

- **玩法**：播放古詩朗讀音頻，玩家需默寫出詩句
- **模式**：
  - 🔤 填字模式：顯示部分字，填入缺少的字
  - 📝 默寫模式：只有音頻，全部自行輸入
- **評分**：字形相似度評分（允許輸入同音字）
- **獎勵**：連續答對觸發連擊獎勵

#### 遊戲三：🔀 文句重組 (Sentence Reassembly)
> **全新設計**

- **玩法**：將打散的文言文句子拖放重新排列
- **難度**：
  - 初級：4-6 個詞語
  - 進階：完整句子（8-12 個詞語）
  - 高級：多句段落重組
- **計時系統**：越快完成越高分
- **提示系統**：使用提示扣分

#### 遊戲四：🎯 字義配對 (Character Matching)
> **全新設計，擴展自 `mensyu2.html` 的測試模式**

- **玩法**：左側文言文字符，右側現代解釋，連線配對
- **格式**：每輪 6 對，限時 60 秒
- **難度遞增**：從常用字到生僻字
- **視覺效果**：正確配對發光動畫，錯誤配對抖動動畫

#### 遊戲五：🏺 古文填空 (Cloze Challenge)
> **整合自 `mensyu2.html` 的測試模式概念**

- **玩法**：文言文段落中有數個空格，從選項中選出正確字詞
- **情境化**：每題都配合對應文豪的著名作品
- **解析**：完成後顯示每題的詳細文法解析

## 四、技術架構 / Technical Architecture

### 4.1 單一 HTML 檔案架構
- **單一 `index.html`** 檔案（自包含，無需伺服器）
- CSS：內嵌 `<style>` 標籤
- JavaScript：內嵌 `<script>` 標籤
- 外部依賴（CDN）：
  - Three.js（3D 遊戲）
  - Font Awesome（圖示）

### 4.2 頁面導航
- 使用 CSS `display: none / block` 切換頁面（無需路由）
- 固定頂部導航欄：首頁 | 文章 | 問答 | 聊天 | 遊戲
- 手機版：底部導航欄

### 4.3 資料儲存
- **localStorage** 儲存：
  - 用戶積分和成就
  - 解鎖角色狀態
  - 閱讀歷史
  - 聊天記錄
  - 遊戲最高分

### 4.4 AI 整合
- **翻譯 API**：Akash Network + DeepSeek-R1（沿用現有實現）
- **古人對話 AI**：相同 API，使用角色特定 system prompt
- 每個角色的 prompt 設定其語言風格和歷史背景

### 4.5 響應式設計
- 桌面：雙欄布局（翻譯區）
- 手機：單欄，底部導航，觸控遊戲控制

## 六、完整使用者旅程範例 / Sample User Journey

> 以下為一位新用戶的完整學習體驗：

1. **進入首頁** → 看到蘇軾（初始解鎖）和其他鎖定角色
2. **點擊蘇軾** → 進入文章選擇（選擇《赤壁賦》）
3. **進入閱讀頁** → 輸入或載入《赤壁賦》原文 → 點擊「翻譯」
4. **AI 翻譯顯示** → 逐段閱讀，點擊不懂的字看解釋
5. **點擊「開始挑戰」** → 進入 5 題問答
   - 回答問題，立即看到對錯反饋
   - 答對 4/5，通過！獲得 +50 分獎勵
6. **解鎖對話功能** → 進入與蘇軾的聊天頁面
   - 蘇軾主動說：「汝剛讀吾之《赤壁賦》，有何感想？」
   - 用戶輸入：「請問您寫赤壁賦的心情是什麼？」
   - 蘇軾以古人語氣回應，並附白話翻譯
7. **完成 3 次對話** → 解鎖遊戲大廳
8. **進入遊戲** → 玩「字義配對」小遊戲
   - 配對與《赤壁賦》相關的文言字詞
   - 獲得 +30 分
9. **查看成就** → 獲得「🌱 初學者」和「📖 勤學者」徽章
10. **繼續挑戰韓愈** → 系統提示：「再完成蘇軾 2 篇文章，即可解鎖韓愈！」
